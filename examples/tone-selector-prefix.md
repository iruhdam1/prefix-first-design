# Example: Tone Selector Prefix

A real prefix from a prototyping session in December 2024.
Four distinct concepts were generated in 45 minutes from this single base.
The prefix never moved — only the exploration did.

---

```
PROBLEM FRAME
Marketing teams at B2B SaaS companies spend 2–3 hours per campaign
writing copy variations for different audience segments. They need
faster ways to generate on-brand copy without losing tone control.

USER
A content manager at a mid-size SaaS company. Comfortable with tools,
not technical. Manages 3–5 campaigns per month, works alone or with
one other person.

CONSTRAINTS (fixed)
Must work within an existing CMS. No new login or account creation.
Output must be editable, not auto-published.

DESIGN PRINCIPLES
Tone control over speed. User always feels in charge of the output.
Reduce blank-page anxiety without removing judgment from the process.

WHAT HAS ALREADY BEEN DECIDED
The primary interaction is a tone selector, not a freeform prompt.
Users pick from preset tones, not write instructions.
```

---

## What this prefix enabled

With this base set once, four concept directions were explored in the same session:

1. A bulk copy generator with a 2x2 grid tone selector based on user choice
2. Tone and intensity slides — the same tones controllable via sliders
3. A spider chart for analysing how a specific copy performs across eight parameters
4. A manual tone selector where users pick tones by directly adjusting the spider chart

The industry and sector context never had to be re-explained between concepts.
Each concept was EXPLORING output tested against the stable prefix.

---

## The breakpoint added mid-session

> "We've decided the primary interaction is a tone selector, not a freeform prompt.
> Users pick from preset tones, not write instructions."

This went into WHAT HAS ALREADY BEEN DECIDED before concept 2.
It meant concept 3 and 4 didn't have to re-establish the interaction model.
